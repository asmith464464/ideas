# Ideas

A static site that documents quantitative ideas as narrative research reports. Strategies are computed locally; results are committed as static artifacts; GitHub Pages renders everything — no computation at build time.

## How it works

```
docs/ideas/<slug>/report.md      # prose narrative with {{ tags }}
docs/ideas/<slug>/config.yaml    # metadata + strategy parameters
docs/ideas/<slug>/artifacts/     # generated by run_research.py — commit these

python run_research.py --idea <slug>   # fetch data, run backtest, write artifacts
python build.py                        # resolve {{ tags }} → _posts/
git push                               # GitHub Actions runs jekyll build + deploy
```

## Adding a new idea

1. Create `docs/ideas/00N-your-idea/` with `config.yaml` and `report.md`
2. Write the narrative using `{{ tags }}` where results will appear
3. Create `research/ideas/00N-your-idea/strategy.py` — implement `fetch_data`, `generate_signals`, `backtest`
4. `python run_research.py --idea 00N-your-idea`
5. `python build.py --idea 00N-your-idea`
6. `bundle exec jekyll serve` — check locally
7. Set `status: published` in `config.yaml`
8. `python build.py`, commit everything including `artifacts/` and `_posts/`, push

## Tag syntax

```
{{ metric: sharpe_ratio }}
{{ metric: max_drawdown_pct | label="Max drawdown" | positive_is_good=false }}
{{ chart: equity_curve | caption="Strategy vs SPY" }}
{{ metric_table }}
{{ signal_table | rows=20 }}
```

Missing artifacts render as placeholders — the site builds cleanly before any computation is run.

## Development setup

```bash
pip install -r requirements.txt
bundle install
```

Run all tests:

```bash
pytest tests/ -v
```

## CLI reference

```bash
python run_research.py --all
python run_research.py --idea 001-time-series-momentum
python run_research.py --tag momentum
python run_research.py --no-cache --idea 001-time-series-momentum

python build.py
python build.py --idea 001-time-series-momentum
python build.py --include-drafts
```
